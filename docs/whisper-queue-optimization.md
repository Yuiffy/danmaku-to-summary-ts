# Whisper 队列优化说明

## 问题描述

之前的晚安回复生成流程存在以下问题:

1. **Whisper锁等待时间配置错误**: 注释说是6分钟,实际是30分钟(180次 × 10秒)
2. **进程超时时间不足**: 处理进程30分钟超时,但Whisper可能等待30分钟,导致超时被强制终止
3. **显存不足时卡死**: 当用户玩游戏或使用显存时,Whisper会卡在加载模型阶段,无法给出明确反馈
4. **日志信息不够详细**: 等待时只有简短的日志,用户不知道具体状态

## 优化方案

### 1. 修正Whisper锁配置 (`enhanced_auto_summary.js`)

**修改内容**:
- ✅ 将最大重试次数从180次增加到360次(60分钟)
- ✅ 修正注释错误(180次×10秒=30分钟,不是6分钟)
- ✅ 增加详细的进度日志:
  - 每10秒输出简短状态
  - 每30秒输出详细队列信息(等待时间、持锁进程、剩余时间等)
- ✅ 在锁文件中记录正在处理的视频文件名
- ✅ 显示等待时长提示

**效果**:
```
⏳ 等待 Whisper 锁释放... (120s)
📊 [Whisper队列状态]
   当前等待时间: 2.0 分钟 (120秒)
   当前持锁进程: PID 12345 (已持有 15.3 分钟)
   当前处理文件: 录制-25788785-xxx.flv
   剩余最大等待: 58.0 分钟
   💡 提示: 如果您正在玩游戏或使用显存，Whisper会等待显存释放
```

### 2. 增加进程超时时间 (`production.json`)

**修改内容**:
- ✅ 将 `processTimeout` 从 30分钟(1800000ms) 增加到 90分钟(5400000ms)

**原因**:
- Whisper锁最多等待60分钟
- 实际处理可能需要10-20分钟
- 总计需要70-80分钟,设置90分钟留有余量

### 3. 添加显存检测 (`check_gpu_memory.py`)

**新增功能**:
- ✅ 检查GPU显存状态(总量、已用、空闲)
- ✅ 判断是否有足够显存(需要4.5GB空闲)
- ✅ 支持等待模式,持续检查直到显存充足或超时
- ✅ 提供详细的状态反馈

**使用方式**:
```bash
# 快速检查模式
python check_gpu_memory.py

# 等待模式(最多等待1800秒)
python check_gpu_memory.py --wait 1800
```

### 4. 集成显存检测到Whisper处理流程 (`batch_whisper.py`)

**修改内容**:
- ✅ 在加载Whisper模型前先检查显存
- ✅ 如果显存不足,等待最多30分钟
- ✅ 给用户明确提示(正在玩游戏/使用显存时)
- ✅ 超时后给出建议(关闭占用显存的程序)

**处理流程**:
```
1. 检查显存状态
   ├─ 充足 → 直接加载模型
   └─ 不足 → 等待显存释放(最多30分钟)
       ├─ 成功 → 加载模型
       └─ 超时 → 退出并提示用户
```

### 5. 持久化队列支持 (`whisper_queue_manager.js`)

**新增功能**:
- ✅ 任务队列持久化到文件 (`.whisper_queue.json`)
- ✅ 支持重启后自动恢复未完成的任务
- ✅ 任务状态跟踪 (pending/processing/completed/failed)
- ✅ 队列统计和可视化显示
- ✅ 自动清理旧任务记录（保留最近100个）

**任务状态**:
- `pending`: 等待处理
- `processing`: 正在处理
- `completed`: 已完成
- `failed`: 处理失败

**重启恢复机制**:
```
1. 程序启动时自动加载队列文件
2. 检测到 'processing' 状态的任务
3. 将其重置为 'pending' 状态
4. 按添加顺序继续处理
```

**队列文件示例** (`.whisper_queue.json`):
```json
{
  "lastUpdate": "2026-02-03T14:37:22.000Z",
  "tasks": [
    {
      "id": "1738574242000-录制-25788785-xxx.flv",
      "mediaPath": "D:\\Videos\\录制-25788785-xxx.flv",
      "roomId": "25788785",
      "addedTime": 1738574242000,
      "status": "pending"
    }
  ]
}
```


## 配置参数说明

### Whisper锁参数 (`enhanced_auto_summary.js`)

| 参数 | 值 | 说明 |
|------|-----|------|
| `WHISPER_LOCK_TIMEOUT` | 3600000ms (60分钟) | 锁文件过期时间 |
| `WHISPER_LOCK_RETRY_INTERVAL` | 10000ms (10秒) | 重试间隔 |
| `WHISPER_MAX_RETRIES` | 360次 | 最大重试次数(60分钟) |
| `WHISPER_PROGRESS_LOG_INTERVAL` | 30000ms (30秒) | 详细日志输出间隔 |

### 进程超时参数 (`production.json`)

| 参数 | 值 | 说明 |
|------|-----|------|
| `processTimeout` | 5400000ms (90分钟) | 处理进程总超时时间 |

### 显存检测参数 (`check_gpu_memory.py`)

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `required_mb` | 4500MB | 所需最小空闲显存 |
| `max_wait_seconds` | 1800秒 (30分钟) | 最大等待时间 |
| `check_interval` | 30秒 | 检查间隔 |

## 使用场景

### 场景1: 正常处理(显存充足)
```
1. 视频录制完成 → FileClosed事件
2. 获取Whisper锁(立即成功)
3. 检查显存(充足)
4. 加载Whisper模型
5. 生成字幕
6. 生成晚安回复
7. 释放锁
```
**耗时**: 10-20分钟

### 场景2: 队列等待(其他视频正在处理)
```
1. 视频录制完成 → FileClosed事件
2. 尝试获取Whisper锁(失败,有其他进程持锁)
3. 等待锁释放(每10秒输出日志)
4. 获取锁成功
5. 检查显存(充足)
6. 加载Whisper模型
7. 生成字幕
8. 生成晚安回复
9. 释放锁
```
**耗时**: 等待时间 + 10-20分钟

### 场景3: 显存不足(用户正在玩游戏)
```
1. 视频录制完成 → FileClosed事件
2. 获取Whisper锁(成功)
3. 检查显存(不足,仅2GB空闲)
4. 等待显存释放(每30秒检查一次)
   - 用户看到提示: "如果您正在玩游戏或使用显存,请稍等或关闭相关程序"
5. 显存释放(用户关闭游戏)
6. 加载Whisper模型
7. 生成字幕
8. 生成晚安回复
9. 释放锁
```
**耗时**: 显存等待时间 + 10-20分钟

### 场景4: 显存长期不足(超时)
```
1. 视频录制完成 → FileClosed事件
2. 获取Whisper锁(成功)
3. 检查显存(不足)
4. 等待显存释放(30分钟)
5. 超时退出
   - 用户看到提示: "显存等待超时,建议关闭占用显存的程序后重试"
6. 释放锁
7. 进程结束(晚安回复未生成)
```
**耗时**: 30分钟(失败)

### 场景5: 重启恢复(程序中断后继续处理)
```
1. 多个视频录制完成，添加到队列
2. 正在处理第1个视频时，程序意外关闭/重启
3. 程序重新启动
4. 自动加载队列文件 (.whisper_queue.json)
5. 检测到中断的任务:
   - 任务1: processing → 重置为 pending
   - 任务2: pending → 保持 pending
   - 任务3: pending → 保持 pending
6. 按顺序继续处理所有待处理任务
7. 完成所有任务
```
**优势**: 
- 不会丢失任务
- 自动从中断处继续
- 无需手动重新提交

## 注意事项

1. **显存需求**: Whisper large-v3-turbo模型需要约3-4GB显存,建议至少保留4.5GB空闲
2. **等待时间**: 如果经常遇到显存不足,建议在不玩游戏时处理视频
3. **进程超时**: 90分钟超时足够处理大部分场景,如果仍然超时,检查是否有异常卡死
4. **锁文件清理**: 如果进程异常退出,锁文件会在1小时后自动过期
5. **队列文件**: 队列文件 `.whisper_queue.json` 会自动保存，不要手动删除，除非确认所有任务已完成
6. **重启安全**: 程序可以随时重启，未完成的任务会自动恢复并继续处理

## 后续优化建议

1. **优先级队列**: 可以为不同房间设置不同的处理优先级
2. **显存预留**: 可以配置为游戏时段自动跳过处理,等到空闲时段再处理
3. **分布式处理**: 如果有多台机器,可以分散Whisper处理负载
4. **模型缓存**: 保持Whisper模型常驻内存,避免重复加载
